# 2026 Q2 执行路线图（精简版）

周期：2026-02-23 ~ 2026-05-17（12 周）

## 1. 季度目标

1. 稳定性：策略可用性 >= 99.5%，重大回滚 <= 1
2. 风险一致性：误报率较基线下降 >= 30%
3. 收益可解释：归因覆盖率 >= 90%
4. 研发效率：变更可追溯且可回滚

## 2. 阶段里程碑

### Phase A（W1-W4）：观测与一致性

- KPI 基线与快照
- 日级看板阈值告警
- 线上/离线一致性校验
- 日回归 + 周回放闭环

### Phase B（W5-W8）：治理与风险

- 参数/模型版本治理
- 变更审批与回滚清单
- 漂移监控与分级动作

### Phase C（W9-W12）：归因与发布闭环

- VaR/ES 例外回测自动报告
- 收益归因流水线
- 灰度放量机制
- 自动回滚阈值与季度复盘

## 3. 每周固定节奏

1. 周一：确认 KPI 与本周实验目标（<=3）
2. 周二：实现与调参（全部绑定实验编号）
3. 周三：回归验证与风险审计
4. 周四：灰度观察与异常处置
5. 周五：复盘、ADR、下周计划

## 4. 周交付硬门槛

1. KPI 快照
2. 风险例外报告
3. 收益归因表
4. 变更与回滚记录
5. ADR 决策记录

## 5. 失败保护

1. 连续两日重大风险阈值触发：冻结新发布
2. 线上/离线偏差超阈：自动降级到稳定版本
3. 漂移告警持续三日：触发强制重训与参数审计
